Library Management System

This system will help librarians manage books, patrons, and lending processes efficiently.

step1: Gather functional requirements - what are the functionality that our system should support
- It should manage a library - library should have books
- It should manage books - add book, remove book
- It should manage patrons profile - add patron, remove patron
- It should manage lending process - patron should be able to lend books from library
- Patron should be charged based on the return date of the book
- Librarian should be able to say if the book is available or someone borrowed it


step2: Non-functional requirements
- Extensibility/ evolvability
- Maintainability
- Scalability
- Modularity
- Low Coupling, High Cohesion

step3: Define Entities - bottom up approach - we will start from the lowest entity first
- Book
- Library
- Patron
- LendingStrategy

step4: Relationship between entities
- Library has Books - aggregation - one to many
- Library has Patrons - aggregation - one to many
